Aurora software: development / communication tool

aurora.exe

aurora.exe is the command line application to communicate with the Aurora console.

Overall features:

aurora.exe provides services related to program transfer and debug. The application can:
  • transfer executable files
  • erase internal flash memory
  • manage external flash content
  • run, pause and reset programs execution
  • view standard sram content
  • get information about hardware and software revisions
Features in red characters are not implemented but planned.

Call syntax:

aurora.exe can be called without arguments to view the help.

The aurora call syntax is the following:
> aurora [-c auto][-b 230400][-l][-e][-r][-x][-v][-tx][-f program.hex]
-c auto
use a specific serial com port. Values can be:
  • Windows: COM0, COM1 ... auto.
  • Linux: ttyUSB0, ..., rfcomm0 ... auto.
  • Mac OS: tty.usbserial-xxx, ..., tty.aurora-xxx ... auto.
Default is auto, all ports are polled until aurora console answers.
-b 230400
set the communication baudrate. Default is 230400.
-l
list available serial ports.
-e
erase internal flash memory.
-r
reboot the DSP.
-x
run program in internal flash memory.
-v
get console hardware and bootloader revision.
-tx
toggle debug led x (1 or 2) state.
-f program.hex
transfer program program.hex in internal flash.

Use cases:

Transfer a program: > aurora -f blasteroid.hex -x This command will try to detect the aurora console and then transfer the blasteroid.hex program to the internal flash. When the transfer is completed, the program will be executed.
Clear internal flash memory: > aurora -e This command will completely erase the content of the internal flash memory.
List available serial ports: > aurora -l This command will list all accessible serial com ports on the computer.
